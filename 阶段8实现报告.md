## 核心改动

### 1. Verifier 增强 - 真实验证目标

### 2. Artifact 验证 - Owner 与 Freshness

**文件**: `packages/tui/src/artifact-evidence-runtime.ts`

**改动**:
- 新增 `validateArtifactFreshness()` 函数
- `hasStructuredArtifactEvidence()` 和 `hasStructuredArtifactEvidenceForPath()` 支持可选的 freshness 验证
- 验证 owner (ownerSessionId/ownerAgentId)
- 验证时间戳 (createdAt vs maxAgeMs)

**关键规则**:
- 预存的非空文件如果不满足 owner 或 freshness 条件会被拒绝
- 默认向后兼容（不传 options 时不检查）

**测试覆盖**:
```
✅ 接受无 freshness 要求的 artifact
✅ 拒绝 stale artifact (超过 maxAgeMs)
✅ 接受 fresh artifact (在 maxAgeMs 内)
✅ 拒绝错误 owner 的 artifact
✅ 接受当前 owner 的 artifact
✅ 多目标场景验证
```

### 3. Windows Shell Adapter - Compound 命令处理

**文件**: `packages/tools/src/index.ts`

**改动**:
- 新增 `CompoundCommandAdapter` 在 adapter registry 最前面
- 新增 `detectHostCompoundCommand()` 函数检测 host-level semicolon
- 排除 PowerShell native 语法（显式 PowerShell 或 PowerShell script）

**适配逻辑**:
```
Host compound: "echo hello; echo world" 
  → blocked (需要 PowerShell 语法或 &&)

PowerShell native: "$x = 1; Write-Output $x"
  → allowed (PowerShell script 语法)

Explicit PowerShell: "powershell.exe -Command '...'"
  → allowed (显式 PowerShell)
```

**测试覆盖**:
```
✅ 阻止 host-level compound 命令
✅ 阻止 git compound 命令
✅ 允许 PowerShell native semicolons
✅ 允许显式 PowerShell 命令
```

## 关键设计亮点

1. **可选验证** - Freshness 验证是可选的，让调用者根据场景决定
2. **早期拦截** - Compound 命令在 adapter 链最前面检测，避免误适配
3. **语法感知** - 明确识别 PowerShell native 语法，不误报
4. **灵活 Owner** - 支持 sessionId 或 agentId 作为 owner 标识
