## 核心功能实现

### 1. Session Memory 在 Compact 时保存和恢复 ✅

**实现位置**: `packages/tui/src/compact-preflight-runtime.ts` (约778行)

**保障机制**:
- **有界**: 最多保存 8 条 session memory
- **脱敏**: 使用 `redactCommonSecrets()` 移除敏感信息
- **截断**: 每条摘要限制 200 字符
- **确定排序**: 保持原始顺序

**恢复机制**: `packages/tui/src/compact-restore-runtime.ts`
- 从 `compactProjection.restoreContext.sessionMemoryRecords` 读取
- 在 "Post-compact restored context" 消息中优先展示
- 不经过磁盘，避免 tombstone 冲突

### 2. 缓存统计分离 main-chain 和 all-calls ✅

**实现位置**: `packages/tui/src/usage-stats-presenter.ts`

**新增字段**:
- `CacheTurnStats.kind`: 标识请求类型（main/continuation/final/agent-child等）
- `CacheSummary.mainChainHitRate`: 主链命中率
- `CacheSummary.allCallsHitRate`: 全部调用命中率

**展示格式**:
```
- main-chain hit rate: 94.2%
- all-calls hit rate: 78.5%
```

### 3. 类型系统扩展 ✅

**CompactRestoreContext**:
```typescript
sessionMemoryRecords: Array<{ id: string; summary: string; scope: string }>;
```

**CacheTurnStats**:
```typescript
kind?: "main" | "continuation" | "final" | "agent-child" | "side-question" | "deep-compact";
```

所有新增字段均为可选，保持向后兼容。

## Memory Tombstone 处理

✅ **无需修改** - Session scope memory 天然豁免 tombstone：
- `isMemoryTombstoned()` 在 session scope 时直接返回 false
- Compact→Resume 过程不经过磁盘，不触发 tombstone 检查
- 设计上安全可靠
